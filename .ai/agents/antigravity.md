# Antigravity Instructions

Use `.ai/skills/angular/` as project-local context for the Angular skills.
Use `.ai/skills/dotnet/` as project-local context for the .NET skills (currently empty).

Before editing code, always read the specific `SKILL.md` file matching the feature area:
- **Architecture**: `.ai/skills/angular/architecture/SKILL.md` (`angular-architecture`)
- **Components**: `.ai/skills/angular/component/SKILL.md` (`angular-component`)
- **Forms**: `.ai/skills/angular/form-component/SKILL.md` (`angular-form-component`)
- **Facades**: `.ai/skills/angular/facade/SKILL.md` (`angular-facade`)
- **Routing**: `.ai/skills/angular/routing/SKILL.md` (`angular-routing`)
- **Services**: `.ai/skills/angular/service/SKILL.md` (`angular-service`)
- **Theming**: `.ai/skills/angular/material-m3-theming/SKILL.md` (`angular-material-m3-theming`)
- **Testing**: `.ai/skills/angular/testing/SKILL.md` (`angular-testing`)

When executing changes, prioritize following these local design patterns (e.g. signal inputs, functional injects, facade controllers, Material 3 override mixins) over standard documentation.
